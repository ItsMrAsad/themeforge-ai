The Problem

The copy-paste workflow
is quietly breaking stores

Merchants customizing Shopify themes today juggle browser tabs, paste unvetted AI output directly into live files, and hope nothing breaks.

⚠

Unoptimized Output

AI chat tools generate working Liquid, rarely optimized Liquid. Pasted code bloats page weight, slows Core Web Vitals, and degrades performance rankings directly.

⛓

Manual & Error-Prone

Merchants copy snippets across tabs and paste them directly into theme files with no preview, no undo path, and no structured validation. One wrong paste can break the live store.

◈

Fragile Theme State

Ad-hoc edits accumulate over time, creating themes that are hard to maintain, impossible to hand off, and fragile to future updates. No history, no baseline, no rollback.

Risk & Mitigation

Key risks, and how
we're de-risking them

Shopify Write Access

Without write access, edits cannot reach the merchant's store. Public App Store apps require a "Page builders" category exemption from Shopify.

Mitigation: Launch on per-merchant custom-app tokens — no exemption needed. Submit the public-app exemption request in parallel. Confirm token write access via a short technical test before building.
Preview Embedding Terms

Embedding the live Shopify preview inside our platform may conflict with Shopify's terms on how store content is presented.

Mitigation: Confirm in writing with Shopify before building. The preview is built as a swappable module so the embedding method can be replaced without rework if the answer changes.
Model Quality & Cost

Edit quality and per-edit cost directly affect product experience and margins. Poor quality erodes merchant trust; high cost breaks unit economics.

Mitigation: Planning model is a swappable component. A/B-test candidates on real Shopify theme edits and route by complexity to control cost and quality independently.
Theme Variety

Shopify themes differ significantly — older legacy architectures vs. modern OS 2.0 themes. Edits that work for one may fail or misfire for another.

Mitigation: Detect the theme's architecture on import and adapt edits accordingly. The Validate stage handles variety by checking every edit before saving — no theme is assumed safe.
